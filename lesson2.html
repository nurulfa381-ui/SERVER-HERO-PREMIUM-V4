Cerita Misi

Taklimat Agent

Server baharu telah berjaya dipasang dengan Windows Server 2019, tetapi server masih belum mempunyai alamat IP yang tetap. Jika server menggunakan alamat IP dinamik, alamat tersebut boleh berubah dan menyebabkan Active Directory, DNS serta DHCP gagal diakses oleh komputer client.

Tugas anda ialah menetapkan alamat IP statik 192.168.10.10, Subnet Mask 255.255.255.0, Default Gateway 192.168.10.1 dan Preferred DNS 192.168.10.10.

Nota Ringkas

Komponen Konfigurasi TCP/IP

IP Address

Alamat unik yang mengenal pasti server dalam rangkaian. Contoh: 192.168.10.10.

Subnet Mask

Menentukan bahagian rangkaian dan bahagian host. Contoh: 255.255.255.0.

Default Gateway

Laluan keluar ke rangkaian lain atau Internet. Contoh: 192.168.10.1.

Preferred DNS

Menukar nama domain kepada alamat IP. Bagi Domain Controller, DNS biasanya menunjuk kepada IP server sendiri.

Kenapa Penting

Mengapa Server Perlu IP Statik?

Alamat Kekal

Client sentiasa dapat mencari server pada alamat yang sama.

DNS Stabil

Rekod DNS tidak rosak akibat perubahan alamat IP.

Active Directory Berfungsi

Domain Controller memerlukan alamat yang konsisten.

Mudah Diuji

Pentadbir boleh melakukan ping dan troubleshooting dengan tepat.

Prosedur

Langkah Konfigurasi Alamat IP Statik

1

Buka Network Connections

Klik kanan ikon rangkaian, pilih Open Network & Internet settings, kemudian buka tetapan adapter.

2

Buka Properties Adapter

Klik kanan Ethernet dan pilih Properties.

3

Pilih Internet Protocol Version 4

Pilih Internet Protocol Version 4 (TCP/IPv4) dan tekan Properties.

4

Masukkan Alamat IP

Masukkan IP Address, Subnet Mask, Default Gateway dan Preferred DNS mengikut pelan rangkaian.

5

Simpan Tetapan

Tekan OK dan tutup semua tetingkap konfigurasi.

6

Uji dengan ipconfig dan ping

Gunakan ipconfig untuk melihat tetapan dan ping 192.168.10.1 untuk menguji gateway.

💡

Tip Penting

IP server dan Default Gateway mesti berada dalam rangkaian yang sama. Bagi rangkaian 192.168.10.0/24, alamat seperti 192.168.10.10 dan 192.168.10.1 berada dalam subnet yang sama.
