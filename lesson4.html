Cerita Misi

Taklimat Agent

Domain serverhero.local telah berjaya diwujudkan, tetapi komputer client masih perlu menukar nama domain kepada alamat IP untuk berkomunikasi dengan server.

Tugas anda ialah mengkonfigurasi DNS Server, membina Forward Lookup Zone, menambah Host (A) Record dan memastikan arahan nslookup server.serverhero.local memulangkan alamat IP yang betul.

Nota Ringkas

Konsep Penting DNS

DNS

Sistem yang menukar nama domain kepada alamat IP.

Forward Lookup Zone

Menukar nama host kepada alamat IP.

Reverse Lookup Zone

Menukar alamat IP kepada nama host.

Host (A) Record

Memetakan nama host kepada alamat IPv4.

NS Record

Menunjukkan Name Server yang berautoriti bagi zone.

FQDN

Nama penuh host seperti server.serverhero.local.

Kenapa Penting

Mengapa DNS Sangat Penting?

Nama Mudah Diingati

Pengguna mengakses server menggunakan nama, bukan nombor IP.

Active Directory

Domain Controller bergantung kepada DNS untuk mencari servis domain.

Name Resolution

DNS memastikan nama host diterjemahkan kepada alamat IP yang tepat.

Troubleshooting

Alat seperti nslookup membantu mengesan masalah DNS.

Prosedur

Langkah Konfigurasi DNS

1

Buka DNS Manager

Dalam Server Manager, pilih Tools dan buka DNS.

2

Buka Forward Lookup Zones

Kembangkan nama server dan klik kanan Forward Lookup Zones.

3

Pilih New Zone

Gunakan New Zone Wizard untuk mencipta zone baharu.

4

Pilih Primary Zone

Pilih Primary Zone dan simpan zone dalam Active Directory.

5

Masukkan Zone Name

Gunakan nama zone serverhero.local.

6

Pilih Secure Dynamic Updates

Benarkan hanya secure dynamic updates untuk keselamatan.

7

Cipta Host (A) Record

Masukkan Host Name server dan IP 192.168.10.10.

8

Uji dengan nslookup

Jalankan nslookup server.serverhero.local.

💡

Tip Penting

Preferred DNS pada Domain Controller sepatutnya menunjuk kepada alamat IP server sendiri, iaitu 192.168.10.10.
