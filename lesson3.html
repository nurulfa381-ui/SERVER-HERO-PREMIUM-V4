Taklimat Agent

Server telah dipasang dan alamat IP statik telah berjaya dikonfigurasikan. Kini organisasi memerlukan satu sistem pusat untuk mengurus pengguna, komputer dan polisi keselamatan.

Tugas anda ialah memasang peranan Active Directory Domain Services (AD DS), mempromosikan server sebagai Domain Controller dan mencipta domain serverhero.local.

Nota Ringkas

Konsep Penting Active Directory

Active Directory

Pangkalan data berpusat yang menyimpan maklumat pengguna, komputer, kumpulan dan sumber rangkaian.

Domain Controller

Server yang mengesahkan log masuk pengguna dan mengurus keselamatan domain.

Domain

Nama logik organisasi dalam rangkaian. Contoh: serverhero.local.

DSRM Password

Kata laluan khas untuk pemulihan Directory Services apabila berlaku kerosakan.

Kenapa Penting

Mengapa Organisasi Memerlukan AD DS?

Pengurusan Berpusat

Pengguna dan komputer boleh diurus dari satu tempat.

Keselamatan

Polisi kata laluan dan akses boleh dikawal secara seragam.

Single Sign-On

Pengguna boleh menggunakan satu akaun untuk banyak sumber.

Skalabiliti

Sesuai untuk organisasi kecil hingga perusahaan besar.

Prosedur

Langkah Pemasangan AD DS

1

Buka Server Manager

Pastikan server menggunakan alamat IP statik sebelum memulakan pemasangan.

2

Pilih Add Roles and Features

Gunakan menu Manage untuk membuka wizard pemasangan peranan.

3

Pilih Role-based or feature-based installation

Pilih server sasaran daripada server pool.

4

Pilih Active Directory Domain Services

Tambah semua ciri yang diperlukan dan teruskan pemasangan.

5

Tekan Install

Tunggu sehingga pemasangan peranan selesai.

6

Promote this server to a domain controller

Klik pautan promosi selepas pemasangan berjaya.

7

Pilih Add a new forest

Masukkan nama domain serverhero.local.

8

Tetapkan DSRM Password

Gunakan kata laluan yang kukuh dan simpan dengan selamat.

9

Jalankan Prerequisites Check

Pastikan tiada ralat kritikal sebelum menekan Install.

10

Restart dan Sahkan Domain

Log masuk menggunakan akaun domain dan semak Active Directory Users and Computers.

⚠️

Kesilapan Biasa

Jangan pasang AD DS sebelum menetapkan alamat IP statik. Perubahan alamat IP selepas server menjadi Domain Controller boleh menyebabkan masalah DNS dan log masuk domain.
